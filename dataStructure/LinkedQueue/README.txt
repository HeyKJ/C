링크드 큐

순환큐화는 다르게 구현이 매우 간편하다.
미리 선언해야 하는 배열과 달리 동적으로 데이터 추가가 가능하기 때문에
후단이 실제 인덱스보다 1뒤에 위치해야 할 필요도, Full 상태와 empty 상태를 구분하기 위해 노드 더미를 추가 할 필요도 없다.

(전단)[노드] - [노드] - 노드 - (후단)[노드]

*데이터 추가
(전단)[노드] - [노드] - 노드 - [노드] - (후단)[노드]

*데이터 삭제
(전단)[노드] - 노드 - [노드] - (후단)[노드]

보는바와 같이 데이터 추가시 후단 노드뒤에 새로운 노드를 연결해주면 되고
데이터 삭제시 전단 노드를 반환하면 된다.

길이를 지정하지 않고 동적으로 할당하기 때문에 Full 개념이 없으므로
isFull에 대한 함수는 존재하지 않는다.
