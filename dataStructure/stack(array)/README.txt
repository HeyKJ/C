스택(배열)
*데이터가 쌓여있는 모양새
*처음에 들어간 것이 나중에 나오는 FILO(First In, Last Out) 또는 LIFO(Last In, First Out)의 구조
*요소의 삽입과 삭제가 자료구조의 한쪽 끝에서만 이루어지는 것이 특징
*자동 메모리, 대부분의 네트워크 프로토콜, 텍스트 분석 프로그램 등 스택 구조로 이루어짐

구조
*1부터 시작된 5개의 자연수 데이터를 예로 들자면

[5] - 가장 나중에 삽입된 데이터 / 가장 처음에 반환될 데이터
[4]
[3]
[2]
[1] - 가장 처음에 삽입된 데이터 / 가장 나중에 반환될 데이터

기능
*삽입
	- 데이터가 순차적으로 삽입되어야 함
*반환
	- 데이터가 배열의 역순으로 반환되어야 함
*위치
	- 현재 배열의 어느 인덱스까지 데이터가 존재하는지 알 수 있어야 함

장점
*배열을 이용하면 구현하기 쉬움
*데이터 접근 속도가 빠름

단점
*동적으로 스택의 용량을 조절하기 힘듬(데이터 최대 개수를 미리 정해야 함)
