수식 트리
=========

수식을 표현하는 이진트리라 해서 수식 이진 트리라고도 불린다.  
수식 트리는 두가지 규칙을 가진다.  

규칙
====
1. 피연산자는 루트 노드이거나 가지 노드이다.
2. 연산자는 루트 노드이거나 가지 노드이다.

즉 leaf부분은 항상 피연산자가 있어야만 한다.  
또한 루트 노드는 연산자만 될 수 있다.  
이미지는 [여기](http://blog.naver.com/PostView.nhn?blogId=yeop9657&logNo=220898006536&parentCategoryNo=&categoryNo=119&viewDate=&isShowPopularPosts=false&from=postView)를 참조하면 나와있다.

알고리즘
========
위의 두가지 규칙에 의거해서 수식 트리를 만들어야 하는 것을 알았다면 이제 후위 표기식을 읽어 수식 트리를 구축하는 알고리즘을 알아본다.  
사실 후위 표기식을 읽어 수식 트리를 구축하는 알고리즘은 여러가지가 있지만 우리는 아래와 같은 알고리즘을 사용한다.  
1. 수식을 뒤에서부터 앞쪽으로 읽어간다.
2. 후위 표기식에서 항상 마지막에 있는 토큰은 연산자이기 때문에 제일 마지막에 있는 토큰이 루트 노드가 된다.  
예를 들어 1 2 + 라는 식이 있을 때, +는 루트 노드가 된다.
3. 그 후 읽어낸 토큰이 연산자일 경우엔 브랜치(가지)가 되며 다음 토큰은 각각 오른쪽 자식과 왼쪽 자식이 된다.  
단 다음 토큰도 연속해서 연산자가 나온 경우 이 토큰의 하위 트리가 완성된 이후 읽어낸 토큰이 왼쪽 자식의 토큰이 된다.
4. 수식에서 읽어낸 토큰이 피연산자인 경우 리프가 된다.
