# 순차 탐색

순차 탐색은 처음부터 시작해서 원하는 값이 나올 때 까지 순차적으로 탐색하는 것을 말한다.  
한 방향으로만 탐색하기 때문에 선형 탐색이라고도 한다.  

## 자기 구성 순차 탐색
순차 탐색은 가장 기본되는 탐색이다.  
링크드리스트로 구성된 데이터 집합을 탐색하고자 할 때 헤드 노드부터 시작하여 찾고자 하는 데이터가 나올 때 까지 다음 노드로 이동한다.  
데이터의 크기가 작으면 성능의 이슈는 크지 않겠지만, 어쨌거나 순차 탐색은 확실하면서도 절대 효율적인 방법은 아니다.  
따라서 자주 쓰이는 데이터 집합을 앞쪽으로 배치하면서 순차 탐색의 효율을 끌어올리는 것이 자기 구성법이다.  
자기 구성법에는 세가지 방법으로 나뉜다.  

1. 전진 이동법(Move To Front)
2. 전위법(Transpose)
3. 빈도 계수법(Frequency Count)

### 전진 이동법(Move To Front)
전진 이동법은 어느 항목이 한 번 탐색되고 나면 데이터 집합의 가장 앞(헤드 노드)으로 위치시키는 방법이다.  
이렇게 하면 다음번에 해당 데이터 집합을 찾을 때 빠르게 탐색을 완료할 수 있다.  
그렇다고 전진 이동법이 모든 경우에 적합한 것은 아니다.  
특정 데이터 집합이 집중적으로 탐색되는 경우는 흔하지 않으므로 신중하게 사용하자.  

### 전위법(Transpose)
전위법은 탐색된 항목을 이전 항목과 위치를 교환한다는 것 말고는 전진 이동법과 동일한 알고리즘이다.  
전진 이동법은 한 번 탐색되고 나면 해당 항목을 가장 앞으로 위치시키지만  
전위법은 점진적으로 앞으로 한 칸씩 이동시킨다. 따라서 자주 탐색되는 항목들을 앞 쪽에 위치시켜 탐색의 효율을 높인 방법이라 할 수 있다.  

### 빈도 계수법(Frequency Count)
빈도 계수법은 각 항목들이 몇 번 탐색되었는지 횟수를 저장해두고 탐색된 횟수가 높은 순으로 데이터 집합을 나열하는 방식이다.  
전위법은 가장 마지막에 위치한 항목이 가장 많은 선택을 받았다 하더라도 앞 쪽에 위치한다는 보장을 받을 수 없지만  
빈도 계수법은 탐색된 횟수를 저장해두어 탐색 횟수 내림차순으로 데이터 집합들이 재구성되기 때문에  
가장 많은 선택을 받은 항목이 가장 앞쪽에 있다는 확실한 보장을 받을 수 있다.  
물론 탐색 횟수를 따로 저장해야하는 메모리 소비나, 탐색이 이루어질 때 마다 데이터 집합이 재구성 되는 비용이 추가적으로 든다.
